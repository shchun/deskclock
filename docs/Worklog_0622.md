# Worklog — 2026-06-22 (Desk Clock)

오늘은 **초음파 소나 기반 사람(움직임) 감지 → 화면 자동 디밍** 기능을 PoC로 설계·구현하고 실기기 2대에서 검증했다. 카메라·WiFi·블루투스를 못 쓰는 제약에서, 스피커로 초음파 톤을 내보내고 마이크로 반사파의 도플러 변화를 분석하는 방식.

## 감지 알고리즘 변천 (핵심 교훈)

1. **사이드밴드/캐리어 비율** — 캐리어가 노이즈 바닥이면 노이즈÷노이즈로 발산 → **항상 MOTION** 오판.
2. **프레임 차분(시간 변화)** — 정지 반사·캐리어 누설은 상쇄돼 누설 오판은 해결. S24에서 정지/움직임 구분 성공. 그러나 **등속(steady) 움직임엔 둔감**(도플러 피크가 안정적이면 프레임 간 변화가 작음) → Fold5 저볼륨에서 "마구 움직여도 안 잡힘".
3. **사이드밴드 절대량 − 노이즈, 캐리어 스커트(±30Hz) 제외** *(현재 코드, 빌드/검증 전)* — 등속 움직임도 잡도록 절대 도플러 에너지 측정. **아직 빌드·실기기 검증 안 함** → 다음 세션 1순위.

부가:
- **신호 가드**: 캐리어 SNR < 6dB면 `NO SIGNAL`(회색) 표시 → 거짓 MOTION 차단.
- **주파수 응답 자가진단(sweep)**: 2~22kHz를 훑어 그 단말이 포착하는 최고 주파수를 찾아 **캐리어 자동보정 + 단말별 `localStorage` 저장**.
- **히스토리**: 5초 버킷 시간축 + 상태 전환 이벤트 로그, 최대 10시간/보관 12시간, `localStorage` 영속, 타임라인 캔버스(시:분 눈금) + 로그 복사.
- 라벨을 **MOTION / NO MOTION**(이진)으로 — 도플러는 '재실'이 아니라 '움직임'만 측정한다는 본질 반영.
- **두 타임아웃 분리**: `무동작 표시 시간`(라벨, 기본 15s)과 `화면 디밍 지연`(실제 밝기, 기본 10분) — detector는 민감하게(15s) 반응, 디밍은 보수적으로(10분) 적용. 움직이면 즉시 풀밝기 복귀.

## 단말별 검증 결과

| 단말 | 캐리어 | 비고 |
|------|--------|------|
| **R5KL307VAST** (SM-S948N / S24 Ultra) | 19.5kHz, −58~−76dB | 19.5kHz 그대로 포착·동작. 정지/움직임 구분 OK. |
| **R3CW70S1DZZ** (SM-F946N / Z Fold5) | 20kHz(자동보정), −29~−72dB | **미디어 볼륨이 0(음소거)** 이라 전 대역 미포착 → 볼륨 올리니 20kHz까지 강하게 포착(SNR 90dB+). |

**규명한 함정들**
- Fold5 "전 대역 −133dB" = **미디어 볼륨 0**. 소나는 미디어 볼륨에 의존(0이면 무음·무신호).
- 볼륨↑ 하니 20kHz 톤이 **가청**(스피커 고출력 시 가청 왜곡 추정). 볼륨↓(미디어 2)로 비가청 확보.
- 그러나 **저볼륨 = 도플러 반사 약함** → Fold5에서 움직임 미검출(현재 미해결). 가청 비가청과 감지 민감도의 **트레이드오프**가 핵심 난제.

## 배터리 (상시 켜둠 대비)

- 절약: OLED라 디밍(최소 밝기+어두운 페이스)로 패널 전력 대폭↓.
- 비용: 소나가 **상시** 마이크+스피커+FFT 소모. 연속 모드 ~100–200mW, **듀티 모드 ~수십 mW**.
- 결론: **듀티 모드 + 자리 비움이 길수록 순절감** 가능. 연속 모드 + 거의 상주면 절약분을 상쇄할 수 있음. 정확치는 `batterystats` 실측 필요.

## 빌드 / 디바이스

- 절차: `npx cap copy android` → `gradlew.bat assembleDebug` → `adb -s <serial> install -r` → 런처 실행. (app/ 변경 시 `cap copy` 필수)
- 진입: 시계 화면 탭 → nav의 `SONAR` 링크 → PoC. 시작 → (NO SIGNAL이면) 설정 → 주파수 응답 측정.
